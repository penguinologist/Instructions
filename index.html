HDITA template for Technical Writing at Virginia Tech (Summer 2014)

Template for Technical Writing HDITA experiment at Virginia Tech

  1. Create your github.io repository following the instructions on the Github Pages site
  2. Download or clone this template (links on the right sidebar of this text)
  3. Save the template to your local username.github.io repository
  4. Update the site's title and information in the _config.yml file
  5. Open and modify my sample files for tasks, concepts, and references
    • See that I have named the concepts as c-something.html, the tasks as t-something.html, and the references as r-something.html. That makes it easier to tell what information type they follow.
  6. Copy the task samples for as many tasks as you need for your project, writing your own content
  7. Pay attention to the YAML metadata on every topic, giving it a unique name
  8. Save your images, as needed, in the images folder. Give them simple names
  9. Update the topics.yml file inside the _data folder to create your map. Copy for as many topics as needed. Avoid tab spaces
  10. Commit your changes to your username.github.io repository
  11. Enjoy your lovely site in big-screen and mobile devices.
